CALCULATION ENGINE — VALIDATED FORMULAS (DO NOT MODIFY)
=================================================================

1. Handicap Differential (per round):
   diff = (score - rating) × (113 / slope)

2. Estimated Handicap:
   Sort all diffs ascending → take best (lowest) up to 8 → average
   estHcp = sortedDiffs.slice(0,8).reduce((s,d)=>s+d,0) / min(8, n)

3. Tier Assignment:
   Find tier whose representative HCP is closest to estHcp
   TIER_ORDER = ['100+','90s','Mid80s','HighSD','Single','Low','Scratch','Plus']

4. Stat Normalization:
   userFW    = totalFwHit / (18 - par3s per round, summed)
   userGIR   = totalGIR / totalHoles
   userPutts = totalPutts / totalGIR
   userScram = totalScrambles / totalMissedGreens

5. Target Ratios:
   ratioFW    = max(0, min(100, 100 × userFW / target.fw))
   ratioGIR   = max(0, min(100, 100 × userGIR / target.gir))
   ratioPutts = max(0, min(100, 100 × target.putts / userPutts)) ← INVERTED
   ratioScram = max(0, min(100, 100 × userScram / target.scramble))

6. Strokes by Category (Excel multiplier method):
   rawD = -3.5 × (max(0, 100 - ratioFW)    / 100)
   rawA = -6.0 × (max(0, 100 - ratioGIR)   / 100)
   rawP = -2.0 × (max(0, 100 - ratioPutts) / 100)
   rawS = -3.5 × (max(0, 100 - ratioScram) / 100)
   scaleFactor = totalGap / (|rawD|+|rawA|+|rawP|+|rawS|)
   strokesX = |rawX| × scaleFactor

NEXT STEPS — DEPLOYMENT ROADMAP
=================================================================

Recommended build order (each = roughly 1 session):

─────────────────────────────────────────────────────
STEP 1: Deploy to Live URL
─────────────────────────────────────────────────────
Goal: Anyone in the world can use TrueStrokes at a real URL.

Tech: Vercel (free) + custom domain (~$15/year)
New concepts: GitHub (version control), git push, environment

What to build:
1. Create GitHub account + repository
2. Push index.html to GitHub
3. Connect GitHub repo to Vercel — auto-deploys on every push
4. Buy domain (truestrokes.com or similar) via Namecheap
5. Point domain at Vercel (DNS settings)
6. Vercel handles HTTPS/SSL automatically

After this step: Live on the internet. Data still in localStorage.

─────────────────────────────────────────────────────
STEP 2: User Accounts + Authentication
─────────────────────────────────────────────────────
Goal: Users have real accounts. Data follows them across devices.

Tech: Supabase (free tier — covers ~50k monthly active users)
New concepts: API calls, database tables, authentication flow,
              environment variables

What to build:
1. Create Supabase project (database + auth in one)
2. Create database tables:
   - users (handled by Supabase Auth automatically)
   - rounds (user_id, date, course, score, par, rating, slope,
             par3s, fairways, gir, putts, scrambles, created_at)
3. Add login/signup screen to app (email + password to start)
4. Replace localStorage calls with Supabase API calls
5. On login from any device → rounds load from database

Add Stripe for payments in this step:
- Stripe handles subscriptions, payments, taxes, payouts
- Decide: free trial (5 rounds?) then $15-20/month
- Never take payments without an LLC formed first (see Legal below)

─────────────────────────────────────────────────────
STEP 3: Real Progress Tracking
─────────────────────────────────────────────────────
Goal: Users see their actual handicap trend over time.

Depends on: Step 2 (needs database + timestamps)

What to build:
1. Save a snapshot on every Calculate:
   - estimated_hcp, fw_pct, gir_pct, putts_per_gir, scram_pct
   - timestamp, current_tier, target_tier
2. Progress Tracking tab pulls real historical snapshots
3. "Improvement" stat = true first-round vs latest comparison
4. Handicap trend chart shows actual data points over time

This is the retention feature. Users come back after every
round to watch their line move.

─────────────────────────────────────────────────────
STEP 4: Practice Plan Detail
─────────────────────────────────────────────────────
Goal: Specific, actionable practice recommendations.

Two options (decide at build time):

Option A — Static plans (free):
  Pre-written drill recommendations per category + tier combo
  e.g. "90s Golfer working on Approach: practice 100-150yd shots..."

Option B — AI-generated plans (tiny cost ~$0.003/generation):
  Use Claude API to generate personalized plan based on:
  - Their specific stats, tier, gap, top priority category
  - Returns 3 specific drills tailored to their game
  Example prompt: "User is a 90s Golfer targeting Single Digit.
  GIR is 22%, target is 46%. Generate 3 specific practice drills."

─────────────────────────────────────────────────────
STEP 5 (Future): Course Data Auto-Populate
─────────────────────────────────────────────────────
Goal: User types course name → rating/slope/par fills in.

Tech: Golf Course API (golfcourseapi.com) ~$30-100/month
  - Search by course name
  - Returns rating, slope, par, hole count
  - Removes friction from data entry form

NOT GPS while playing — that is a separate, much larger feature.

GPS / COURSE FEATURE DECISION FRAMEWORK
=================================================================

Near term (next 6 months):
  Course name auto-populate via Golf Course API
  → Removes friction, high user value, ~$30-100/mo

Medium term:
  Arccos / ShotScope import integration
  → Their users can import round data automatically
  → You get data without building GPS infrastructure
  → Positions TrueStrokes as analytics layer, not a tracker

Long term (v2 — only if user demand proves it):
  Native mobile app with GPS
  → Requires licensed course coordinate data ($500+/mo at scale)
  → Separate product decision, not a web app feature

Key insight: Don't compete with Arccos on GPS tracking.
Compete on analytics, coaching voice, and tier identity.
Integration beats replication.

=================================================================
COMPETITIVE LANDSCAPE
=================================================================

Main competitors:
- Arccos: Hardware sensors + GPS + strokes gained. $130 hardware + sub.
- ShotScope: Similar to Arccos. GPS watch focus.
- GolfMetrics: Original strokes gained app by Mark Broadie. ~$70/yr.
- Circles: Used by Tour pros. Expensive. Complex.
- DECADE Golf: Course management system. Not a stats tracker.

TrueStrokes differentiator:
  "You are a 90s Golfer. Here's exactly what it takes to become
   a Single Digit." — Tier identity + gap framing is unique.
  No hardware required. Pure analytics and coaching layer.
  Simpler and more motivating than raw strokes gained numbers.

=================================================================
BUSINESS METRICS TO TRACK
=================================================================

The one number that matters most:
  → Rounds entered per user per month
  If users enter rounds consistently = getting value = will pay
  If users enter one round and disappear = retention problem

Secondary metrics:
  → PDF reports downloaded per user (engagement signal)
  → % of users who enter 5+ rounds (activation threshold)
  → Churn rate once charging (how many cancel per month)

Pricing validated with 4 real test users:
  All said they would pay $15-20/month
  Key retention concern: what keeps them coming after reaching goal?
  Answer: progress tracking + updated targets + social features (later)

=================================================================
TECH STACK DECISION LOG
=================================================================

Hosting: Vercel (chosen over Netlify)
  Reason: Better developer experience, more widely used by startups,
  easier to migrate away from if needed. Same price (free).

Database + Auth: Supabase (chosen over Firebase, PlanetScale, AWS)
  Reason: Built on PostgreSQL (scales to millions of users),
  includes auth system, free tier is generous, simple to use.
  Risk: Venture-backed company still maturing — but since it's
  standard PostgreSQL underneath, migrating away is straightforward.

Current stack: Vanilla HTML + CSS + JS (no framework)
Future consideration: When the app splits into multiple pages
  and gets more complex, consider React or Vue. Not needed yet.

Payments: Stripe
  Reason: Industry standard, handles subscriptions + taxes + payouts,
  2.9% + $0.30 per transaction, no monthly fee until meaningful scale.
