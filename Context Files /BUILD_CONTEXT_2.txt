CALCULATION ENGINE — VALIDATED FORMULAS (DO NOT MODIFY)
=================================================================

These formulas are locked. All 4 test scenarios match Excel exactly.

1. Handicap Differential (per round):
   diff = (score - rating) × (113 / slope)

2. Estimated Handicap:
   Sort all diffs ascending → take best (lowest) up to 8 → average them
   estHcp = sortedDiffs.slice(0,8).reduce((s,d)=>s+d,0) / min(8, numRounds)

3. Tier Assignment:
   Find tier whose representative HCP is closest to estHcp
   TIER_ORDER = ['100+','90s','Mid80s','HighSD','Single','Low','Scratch','Plus']

4. Stat Normalization:
   userFW    = totalFwHit / (18 - par3s per round, summed)
   userGIR   = totalGIR / totalHoles
   userPutts = totalPutts / totalGIR  (only on GIR holes)
   userScram = totalScrambles / totalMissedGreens

5. Target Ratios (progress % toward target):
   ratioFW    = max(0, min(100, 100 × userFW / target.fw))
   ratioGIR   = max(0, min(100, 100 × userGIR / target.gir))
   ratioPutts = max(0, min(100, 100 × target.putts / userPutts))  ← INVERTED (lower is better)
   ratioScram = max(0, min(100, 100 × userScram / target.scramble))

6. Strokes by Category (Excel multiplier method):
   rawD = -3.5 × (max(0, 100 - ratioFW)    / 100)   ← Driving
   rawA = -6.0 × (max(0, 100 - ratioGIR)   / 100)   ← Approach
   rawP = -2.0 × (max(0, 100 - ratioPutts) / 100)   ← Putting
   rawS = -3.5 × (max(0, 100 - ratioScram) / 100)   ← Short Game
   scaleFactor = totalGap / (|rawD|+|rawA|+|rawP|+|rawS|)
   strokesX = |rawX| × scaleFactor

UX PRINCIPLES (maintain throughout)
=================================================================

1. "Sticky" results page — users should want to come back after every round
2. Before/After framing — current tier baseline = "before", you = "now"
3. Above/Below average comparisons are motivating for golfers
4. Progress % toward target is more important than raw stat values
5. Screenshots — Round vs. Tiers should be shareable
6. Coach voice — results speak directly to the golfer, not like a dashboard
7. Mobile friendly — sections stack on small screens
